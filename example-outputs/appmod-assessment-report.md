## Issues — Detail

### 🔴 Mandatory

#### 1. Replace `System.Data.SqlClient` with `Microsoft.Data.SqlClient`

- **Files:** `legacy-code/CustomerRepository.cs` (line 3, 25, 58, 77),
  `legacy-code/OrderProcessor.cs` (line 3, 50, 105),
  `legacy-code/Web.config` (line 7, `providerName`)
- **Why it matters:** `System.Data.SqlClient` is in maintenance mode and does
  not support Azure AD / managed identity authentication. Required for all
  Azure SQL targets.
- **Recommendation:** Predefined task — **"Migrate to Managed Identity based
  Database on Azure"**.
- **Doc:** [SqlClient upgrade scenario](https://learn.microsoft.com/en-us/dotnet/azure/migration/appmod/overview#scenarios)

#### 2. Move `MerchantSecret` (and rotate it) to Azure Key Vault

- **Files:** `legacy-code/Web.config` (line 14, `MerchantSecret`),
  consumed by `legacy-code/PaymentService.cs` (line 23, 85)
- **Why it matters:** Cloud-hosted apps must not ship secrets in source-controlled
  config. Blocks security review on every Azure target. The placeholder value
  in this sample also looks like a real `sk_live_*` token shape — rotate.
- **Recommendation:** Predefined task — **"Migrate to secured credentials by using
  Managed Identity and Azure Key Vault"**.

#### 3. Replace `Integrated Security=True` connection string with managed identity

- **File:** `legacy-code/Web.config` (line 6)
- **Why it matters:** Windows-integrated auth doesn't work from `AppService.Linux`,
  `ACA`, or `AKS.Linux` against Azure SQL. The same predefined task that fixes
  finding #1 covers this — switch to `Authentication=Active Directory Managed Identity`.
- **Recommendation:** Predefined task — **"Migrate to Managed Identity based
  Database on Azure"**.

#### 4. Replace `System.Diagnostics.EventLog`

- **Files:** `legacy-code/OrderProcessor.cs` (line 200),
  `legacy-code/PaymentService.cs` (lines 64, 70, 117) — 4 incidents
- **Why it matters:** `EventLog` is unavailable on `AppService.Linux`, `ACA`, and
  `AKS.Linux`. Even on Windows targets, it bypasses cloud diagnostics.
- **Recommendation:** Predefined task — **"Migrate to OpenTelemetry on Azure"**;
  use `ILogger` with the Application Insights exporter.

#### 5. Upgrade from .NET Framework 4.5

- **File:** `legacy-code/Web.config` (line 27, `targetFramework="4.5"`)
- **Why it matters:** .NET Framework 4.5 is end-of-life. Every modern Azure
  compute target (App Service Linux, ACA, AKS Linux) requires .NET 8+ or
  containerized .NET Framework on Windows nodes.
- **Recommendation:** Scenario — **".NET version upgrade"** (target .NET 8 or later).

### 🟠 Potential

#### 6. Replace `HttpWebRequest` with `HttpClient`

- **File:** `legacy-code/PaymentService.cs` (lines 36, 93) — 2 incidents
- **Why it matters:** `HttpWebRequest` is obsolete; no built-in retry,
  circuit-breaker, or DI integration. Hard to mock for tests.
- **Recommendation:** Use `IHttpClientFactory` + Polly for resilience.

#### 7. SMTP via `System.Net.Mail.SmtpClient` on port 25

- **File:** `legacy-code/OrderProcessor.cs` (line 184),
  `legacy-code/Web.config` (line 18, `SmtpPort=25`)
- **Why it matters:** `SmtpClient` is documented as not recommended for new
  development; outbound port 25 is blocked on most Azure compute, and the
  internal SMTP host (`smtp.contoso.internal`) is unreachable from Azure.
- **Recommendation:** Predefined task — **"Migrate to Azure Communication
  Service email"**.

#### 8. Configuration via `ConfigurationManager.AppSettings` / `ConnectionStrings`

- **Files:** `OrderProcessor.cs` (lines 21, 175, 177–179),
  `CustomerRepository.cs` (line 17),
  `PaymentService.cs` (lines 14, 17, 23, 85)
- **Why it matters:** No environment separation, no hot-reload, no validation,
  no Key Vault integration.
- **Recommendation:** `IConfiguration` + `IOptions<T>` with Azure App
  Configuration / Key Vault providers.

#### 9. No retry / circuit-breaker on outbound calls

- **File:** `legacy-code/PaymentService.cs` (lines 36, 93)
- **Why it matters:** Cloud networks have transient failures; the current code
  surfaces these as user-visible errors.
- **Recommendation:** Add Polly policies via `IHttpClientFactory`.

  with containers.

  — start there.
