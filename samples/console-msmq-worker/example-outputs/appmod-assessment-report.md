## Issues — Detail

### 🔴 Mandatory

#### 1. Replace MSMQ (`System.Messaging`) with Azure Service Bus

- **Files:** `QueueProcessor.cs:4, 10, 18–25, 30–47`
- **Why:** `System.Messaging` was removed from .NET Core / .NET 5+. No shim.
- **Recommendation:** Predefined task — **"Migrate to Azure Service Bus"**.

#### 2. Replace `System.Data.SqlClient` and `Integrated Security=True`

- **Files:** `QueueProcessor.cs:2, 51`, `App.config:4` (`providerName`),
  `App.config:5` (`Integrated Security=True`)
- **Recommendation:** Predefined task — **"Migrate to Managed Identity based
  Database on Azure"**.

#### 3. Replace `System.Diagnostics.EventLog`

- **File:** `QueueProcessor.cs:67–69`
- **Recommendation:** Predefined task — **"Migrate to OpenTelemetry on

#### 4. Upgrade from .NET Framework 4.5

### 🟠 Potential

#### 5. Synchronous Main loop with manual shutdown

- **Files:** `Program.cs:9–43`
- **Recommendation:** Convert to `IHost.RunAsync` + a `BackgroundService`
  whose `ExecuteAsync` awaits a `ServiceBusProcessor`. Drops the manual
  `ManualResetEventSlim` and `Console.CancelKeyPress` plumbing.

#### 6. No retry / DLQ on poison messages

- **File:** `QueueProcessor.cs:30–47, 51–66`
- **Why:** Any deserialization failure or DB exception will cause the
  message to be re-received indefinitely. Service Bus has built-in
  `MaxDeliveryCount` and DLQ — use them.

#### 7. `XmlMessageFormatter` with `[Serializable]` `OrderEvent`

- **Files:** `QueueProcessor.cs:25, 49`, `QueueProcessor.cs:79–86`
- **Why:** Implicit binary-ish contract; brittle on the receiver. Service Bus
  uses byte payloads — use `System.Text.Json` with a versioned schema
  (`v` field).

## Recommended Path

1. Run **"Migrate to Azure Service Bus"** — replace MSMQ. This is the lever
   that unlocks everything else; the new payload contract feeds the
   refactor of `OrderEvent`.
2. Run **"Migrate to Managed Identity based Database on Azure"**.
3. Run **"Migrate to OpenTelemetry on Azure"**.
4. Run **".NET version upgrade"** — converts to .NET 8 console + `IHost` +
   `BackgroundService`. Drop the manual `Main` loop.
5. Containerize and deploy to **Azure Container Apps** with replica scaling
   based on Service Bus queue length.

> 💡 Compared to Sample 3 (Windows Service), this archetype lands on ACA
> faster because there's no `ServiceBase` / SCM coupling — just `Main(...)`.
> If your team has both archetypes, do the console workers first.
