## Issues — Detail

### 🔴 Mandatory

#### 1. `System.ServiceModel` (WCF) — not first-class on Linux compute

- **Files:** `IBillingService.cs:2`, `BillingService.svc.cs:7`,
  `Web.config:23–47` (entire `<system.serviceModel>` section)
- **Why it matters:** WCF runtime is .NET Framework only. The contract,
  bindings, MEX endpoint, and behaviors must all be re-implemented.
- **Recommendation:** Two options —
  - **CoreWCF** for minimum-change port to .NET 8.
  - **ASP.NET Core minimal API** (REST/JSON) or **gRPC** for clean break.
- **Doc:** [.NET version upgrade scenario](https://learn.microsoft.com/en-us/dotnet/azure/migration/appmod/overview#scenarios)

#### 2. Replace `System.Data.SqlClient` with `Microsoft.Data.SqlClient`

- **Files:** `BillingService.svc.cs:4, 18, 50, 79, 105`,
  `Web.config:7` (`providerName`)
- **Why it matters:** No managed-identity support; required for Azure SQL.
- **Recommendation:** Predefined task — **"Migrate to Managed Identity based
  Database on Azure"**.

#### 3. SQL injection in `CreateInvoice`

- **File:** `BillingService.svc.cs:24` (currency concatenated into `INSERT`)
- **Why it matters:** Live vulnerability. Even before migration this should
  be fixed.
- **Recommendation:** Parameterize — add `@cur` parameter.

#### 4. Move `BillingApiKey` to Azure Key Vault

- **File:** `Web.config:11`
- **Why it matters:** Plaintext API key in source-controlled config.
- **Recommendation:** Predefined task — **"Migrate to secured credentials by
  using Managed Identity and Azure Key Vault"**.

#### 5. Replace `Integrated Security=True` connection string

- **File:** `Web.config:6`
- **Why it matters:** Windows-integrated auth doesn't reach Azure SQL from
  Linux compute.
- **Recommendation:** `Authentication=Active Directory Managed Identity`.

#### 6. Replace `System.Diagnostics.EventLog`

- **File:** `BillingService.svc.cs:114, 144` — 2 incidents
- **Why it matters:** Unavailable on `AppService.Linux`, `ACA`, `AKS.Linux`.
- **Recommendation:** Predefined task — **"Migrate to OpenTelemetry on Azure"**.

#### 7. Upgrade from .NET Framework 4.5

- **File:** `BillingService.csproj:13` (`<TargetFrameworkVersion>v4.5</TargetFrameworkVersion>`)
- **Why it matters:** End of life. Required for any modern Azure compute.
- **Recommendation:** Scenario — **".NET version upgrade"** (.NET 8 or later).

**Reading the table:**

- Linux targets are effectively a **rewrite** because of WCF + `System.Web`
  hosting. Even after the rewrite, every *other* Mandatory item (managed
  identity, secrets, EventLog, .NET version, SQL injection) must still be
  fixed.
- `AKS.Windows` lets you lift the WCF service onto Windows nodes, but you
  still owe Microsoft.Data.SqlClient + Key Vault + the SQL injection fix
  before going live. This is the cheapest path to "running in Azure" and the
  most expensive path long-term.

---

## Recommended Path

1. **Decide rewrite vs. lift.** If staying on WCF for >12 months, lift to
   `AKS.Windows`. Otherwise, plan the CoreWCF or REST rewrite now.
2. Fix the SQL injection in `CreateInvoice` *first* — it's a live issue.
3. Run **"Migrate to secured credentials by using Managed Identity and Azure
   Key Vault"** to remove `BillingApiKey` and the SQL credential.
4. Run **"Migrate to Managed Identity based Database on Azure"**.
5. Run **"Migrate to OpenTelemetry on Azure"**.
6. Run **"Migrate to Azure Communication Service email"**.
7. Then the **".NET version upgrade"** scenario (with the chosen WCF strategy).
